# 🇸🇦 Arabic copy for Claude design (`index.ar.html`) — RTL

**How to use this**
1. In Claude design, duplicate the English version.
2. Replace each English string with the Arabic below — section by section.
3. **Set the page language to Arabic and direction to RTL** (right-to-left). *(The build also forces `lang="ar" dir="rtl"` automatically, but set it in the design so the preview mirrors correctly.)*
4. **Flip Arabic to live in the switcher** — in the `langs` array change:
   `{ code: 'ar', label: 'العربية', live: true },`  *(it's currently `live: false` / "soon")*
5. Export and save in the repo root as **`index.ar.html`**.
6. Commit & push to `main`. The pipeline auto-detects it → builds **`/ar/`** with `dir="rtl"`, hreflang + sitemap. **No pipeline change needed.**

**Leave untouched / keep in Latin script (LTR):** brand & product names (Yelo Sale, Qoralia, KhebraOS, Ezhal, RevealSite, Compass Med…), tech names (Laravel, Next.js, FastAPI, Flutter, Redis, AWS…), URLs, emails, phone, and metric tokens (`<200ms`, `12+`, `16+`, `25,000+`). Browsers handle the bidi mixing automatically inside RTL text.

> ⚠️ **RTL design check (important):** Arabic flips the whole layout. In Claude design, verify the **nav, alignment, paddings, and any arrows/chevrons (→, ↗, ↓)** mirror correctly. Directional arrows like `→` ("Read the case study →") usually should point **left** in RTL (←). The pipeline sets `dir="rtl"`; visual polish is design-side.

- الأثر: **مزايدة حيّة** / **مزادات WEBSOCKET** — `<200ms` / **زمن استجابة المزايدة الحيّة** — **ERP** / **مزامنة DYNAMICS 365**
  - → **مزايدة حيّة فورية عبر WebSocket + مزايدة وكيلة تلقائية**
  - → **مدفوعات Amazon Payfort · رمز تحقق GoInfinito · إشعارات FCM**
  - → **توطين عربي/إنجليزي مع دعم RTL كامل**
- المشكلة: **كانت إعادة تسويق المركبات في الخليج تعمل عبر قنوات غير شفّافة خارج الإنترنت — يحرّك أصحاب الأساطيل والوكلاء المخزون عبر صفقات مغلقة، دون شفافية، دون منافسة فورية، مع فوترة تُطابَق يدويًا مع نظام ERP مؤسسي. الهدف: استبدالها بدار مزادات رقمية شفّافة وفورية عبر الويب والجوّال، حيث يستطيع أي مزايد موثَّق المنافسة حيًّا — دون خسارة مزايدة واحدة بسبب تعارض في التزامن.**
- الجزء الصعب: **المزادات الحيّة لا تسامح. حين يطرق عشرات المزايدين الدُّفعة نفسها في الثواني الأخيرة، يجب أن يكون النظام المصدر الوحيد للحقيقة: كل مزايدة مرتّبة بشكل صحيح، دون تحديثات مفقودة، دون «فائزَين»، ومزايدات وكيلة تُحلّ فورًا — منعكسةً على الويب والجوّال فوريًّا، بالعربية والإنجليزية (RTL)، ومتزامنة تمامًا مع نظام Microsoft Dynamics 365 ERP.**
- المعمارية:
  - **عقد واحد، ثلاثة عملاء** — **مستودع Nx أحادي مع واجهة برمجية Laravel 11 تخدم تطبيق ويب Next.js 14 وتطبيق جوّال Flutter (BLoC)، بحيث يتشارك الويب وiOS وAndroid الأنواع والسلوك ذاتها.**
  - **مزايدة فورية بسلطة الخادم** — **حالة المزايدة الحيّة في Redis بزيادات ذرّية؛ وطبقة Socket.IO/Node تبثّ لكل مراقب. الخادم، لا العميل، يقرّر ترتيب المزايدات — ما يلغي تعارضات التزامن والمزايدات المفقودة تحت الحِمل. زمن استجابة <200ms من الطرف إلى الطرف.**
  - **مزايدة وكيلة تلقائية** — **يضع المزايدون سقفًا أقصى؛ ويقابل المحرّك المنافسين تلقائيًّا حتى ذلك السقف، من جهة الخادم، ليفوزوا بأدنى سعر لازم.**
  - **المخزون كآلة حالات** — **تمرّ كل دُفعة عبر: مُدرجة → مزاد حيّ → محجوزة → مدفوعة → مُسلَّمة، مع حوكمة إدارية في كل خطوة.**
  - **وصول بثلاثة مستويات** — **مسؤول / شركة / عميل، كلٌّ خلف حارس JWT خاص به (RBAC متعدّد الحُرّاس).**
  - **مزامنة مؤسسية ومدفوعات** — **مزامنة ثنائية الاتجاه مع Dynamics 365 ERP للعملاء والمركبات والفواتير؛ مدفوعات Amazon Payfort؛ رمز تحقق GoInfinito + تفويض KYC للمزايدين؛ إشعارات FCM؛ فواتير PDF/QR؛ AWS S3.**
- النتيجة:
  - **دار مزادات شفّافة وفورية، حيّة على الويب وiOS وAndroid (App Store + Google Play).**
  - **مزايدة حيّة فورية بزمن استجابة <200ms، مع مزايدة وكيلة تلقائية وصفر مزايدات مفقودة بسبب تعارضات التزامن.**
  - **استُبدلت إعادة التسويق غير الشفّافة خارج الإنترنت بخطّ رقمي من الإدراج حتى التسليم، بفوترة متزامنة مع ERP.**
  - **أُطلق عبر تدفّق trunk-based: alpha/beta/prod (Docker، GitHub Actions CI/CD، AWS).**
- دوري: **مهندس برمجيات — الـ Backend بـ Laravel، والـ Frontend بـ Next.js، وتطبيق الجوّال بـ Flutter.**
